Arrays and Slices in Go (Golang)

Introduction

Arrays and slices are fundamental data structures in Go for storing sequences of elements. Each has its own characteristics and use cases depending on the needs of your program.

Arrays

Definition

An array in Go is a fixed-size sequence of elements of the same type. Once defined, the size of an array cannot be changed.

Example

package main

import "fmt"

func main() {
    // Declaring and initializing an array of integers
    var arr [5]int
    arr[0] = 10
    arr[1] = 20
    arr[2] = 30
    arr[3] = 40
    arr[4] = 50

    fmt.Println("Array:", arr)
}

Key Features

  • Fixed size specified at compile-time
  • Direct access to elements using index positions
  • Useful when you know the exact number of elements needed

Slices

Definition

A slice in Go is a dynamically-sized, flexible view into the elements of an array. Slices are built on top of arrays and provide more flexibility compared to fixed-size arrays.

Example

package main

import "fmt"

func main() {
    // Creating and initializing a slice of integers
    slice := []int{10, 20, 30, 40, 50}

    fmt.Println("Slice:", slice)

    // Modifying the slice
    slice = append(slice, 60)
    fmt.Println("Updated Slice:", slice)
}

Key Features

  • Dynamic size that can change during runtime
  • Efficient appending and slicing operations
  • Reference to an underlying array

Usage Guidelines

Understanding when to use arrays versus slices is essential:

  • Use arrays when you need a fixed-size collection of elements and know the exact size at compile-time.
  • Use slices when you need a dynamic or resizable collection of elements, efficient appending, or when working with parts of arrays.
