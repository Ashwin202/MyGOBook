Maps in Go

Introduction

Maps in Go are a collection of unordered pairs of key-value pairs. They provide an efficient way to store and retrieve data based on keys.

Basic Map Example

Definition and Initialization

A map in Go is declared using the syntax map[keyType]valueType. You can initialize a map using the make function or using a map literal.

package maps

import "fmt"

func Maps() {
    // Declaring and initializing a map
    myMap := make(map[string]int)
    myMap["one"] = 1
    myMap["two"] = 2

    // Deleting an entry from the map
    delete(myMap, "one")

    // Accessing a value and checking if it exists
    val, isFound := myMap["one"]

    // Printing the map and results
    fmt.Println(myMap)
    fmt.Println(myMap["one"], val, isFound)

    // Iterating over a map
    for key, value := range myMap {
        fmt.Println(key, value)
    }

    // Initializing a map with a struct{} type for set-like behavior
    mapStruct := map[string]struct{}{"s1": {}, "s2": {}, "s3": {}, "s4": {}}
    _, ok := mapStruct["s1"]
    fmt.Println(mapStruct, ok)
}

Explanation

  • Definition and Initialization: Shows how to declare, initialize, and modify a map in Go.
  • Deleting Entries: Demonstrates how to delete an entry from a map using the delete function.
  • Accessing Values: Shows how to access values from a map and check if a key exists.
  • Iterating Over a Map: Illustrates how to iterate over key-value pairs in a map using a for loop with range.
  • Map with Struct{}: Initializes a map with struct{} to mimic set-like behavior for fast lookups.

Additional Features of Maps in Go

Concurrency Safety

Go maps are not safe for concurrent use by multiple goroutines by default. To safely access maps concurrently, you should use synchronization mechanisms like channels or sync primitives.

Nil Maps

A nil map behaves like an empty map when reading, but attempts to write to a nil map will cause a runtime panic.

Performance Considerations

Maps in Go provide average-case constant-time complexity for lookups, inserts, and deletes, making them efficient for most use cases.
